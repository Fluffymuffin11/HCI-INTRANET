Change control & deployment

No CI/CD pipeline is configured. Every change flow below ends with an operator on the VM running git pull followed by the listed restart command. Adding a CI/CD job is on the backlog.

Change typeHow it flows
Backend code changegit → PR review → merge → SSH to VM → git pull && docker compose restart backend
Frontend changegit → PR → npm run build → commit dist/ → merge → SSH to VM → git pull (nginx picks it up on next request)
Schema changeEdit prisma/schema.prisma → prisma migrate dev → commit migration → merge → SSH to VM → git pull && docker compose restart backend (prisma migrate deploy runs on boot)
Dependency addEdit app/package.json → merge → SSH to VM → git pull && docker compose restart backend (npm install runs on boot)
nginx config changeEdit nginx/default.conf → merge → SSH to VM → git pull && docker compose restart web
OS / kernel updateScheduled hospital maintenance window → sudo dnf upgrade → reboot
TLS cert renewalAnnual; hospital PKI ticket 30 days before expiry → swap files in /etc/nginx/tls/ → docker compose restart web
